SciGate Help

Getting Started
⚡
Quick start: Enter a GitHub repo (e.g. owner/repo) or a local path in the prompt and click Scan. SciGate will analyze the repository across 6 reproducibility dimensions and return a score from 0 to 100.
Dashboard Panels
◎ Score Gauge
Overall reproducibility score (0–100) with letter grade. The gauge animates on each scan with color coding: teal for good, amber for fair, red for poor.
▦ Dimensions
Six scored dimensions: Environment, Seeds, Data Provenance, Documentation, Testing, and Compliance. Each shows deduction reasons.
⚙ Recommended Fixes
AI-generated fixes ranked by ROI — showing recoverable points and estimated effort. Fixes never touch model/training code.
★ Org Memory
Recurring failure patterns detected across all scanned repositories. Shows frequency and trend direction (rising/falling).
↻ Activity Feed
Recent pull requests and commits from the scanned GitHub repository. Shows PR status, commit SHAs, and authors.
⚒ Dependencies
Dependency health analysis: pin ratio, known CVEs, deprecated packages, and overall health grade (A–F).
✎ Compliance Audit
Checks your repo against an industry compliance profile (Enterprise Production, ML Reproducibility, or Open Source Release). Select a standard, click Audit, and see a pass/fail checklist.
■ Merge Gate
Shows whether the repo score meets the configured threshold (default: 75). Blocked repos cannot auto-merge in CI.
🔒 Secrets
Digs through commit history for deleted or reverted credentials (API keys, tokens, private keys, connection strings). Also scans live files.

Repo Map
🗺
The Repo Map auto-generates a full structural overview of any repository after scanning. Click the Map button in the header (or press M) to open it. It shows:
▦ Language Breakdown
Color-coded bar showing the percentage of each programming language, with file counts and sample files.
📁 Directory Tree
Visual tree of the repo's directory structure with file counts per directory.
★ Key Files
Detects README, LICENSE, Dockerfile, requirements.txt, CI configs, DVC, MLflow, citation files, and more.
⚠ AI Config Detection
Flags AI editor config files (.cursorrules, CLAUDE.md, mcp.json, etc.) that could be repo poisoning attack vectors.
AI Config / Repo Poisoning Detection
⚠
Inspired by Medusa Security Scanner, SciGate detects AI editor configuration files that are known repo poisoning vectors. These files can inject malicious instructions into AI coding assistants (Cursor, Cline, Copilot, Claude Code, Gemini, etc.).

Credential History Digger
🔒
The Secrets panel scans both current files and deleted/reverted commits for leaked credentials. Even if a secret was committed and then removed, it remains in git history and can be extracted by attackers. SciGate flags these so you can rotate the credentials immediately.

About
★
SciGate v2.1 — Scientific Reproducibility Intelligence Platform.
A 5-agent pipeline (Audit, Fix, Memory, Regression, Notify) that assigns a defensible 0–100 reproducibility score, generates targeted AI-authored fixes, enforces org-level quality gates, and learns continuously from every scan. 100% open-source infrastructure — no vendor lock-in.
